Как выложить сайт до покупки домена

Сайт уже статический: его можно загрузить на Netlify, Vercel, GitHub Pages или обычный хостинг. Домен подключается потом.

Панель готовностиДоменный чеклист

Лучший путь для теста

  1. Распаковать архив.
  2. Загрузить папку на Netlify Drop или GitHub Pages.
  3. Проверить все страницы на временном адресе.
  4. Подключить форму заявки.
  5. После проверки купить домен и прописать DNS.

Netlify Drop

Самый быстрый вариант без терминала: перетащить папку проекта в интерфейс Netlify Drop и получить временный адрес.

1. Открыть Netlify Drop
2. Перетащить папку prizhivetsya_pre_domain_v9
3. Получить временный URL
4. Проверить формы и ссылки
5. Позже подключить домен

GitHub Pages

Хорошо, если нужен контроль версий. Подходит для статического сайта без backend.

1. Создать репозиторий
2. Загрузить файлы
3. В Settings → Pages выбрать ветку
4. Проверить адрес вида username.github.io/project
5. После домена настроить custom domain

Что нельзя делать перед проверкой

  • Не включать платные заявки без финальных юридических текстов.
  • Не подключать аналитику без политики cookies и ПДн.
  • Не писать “гарантированно приживётся”.
  • Не публиковать сорта как проверенные, если у них нет источника.
Важно: рекомендации носят информационный характер и не гарантируют приживаемость, урожай или успешную перезимовку. Перед публичным запуском требуется проверка источников и юридических текстов под владельца сайта.
