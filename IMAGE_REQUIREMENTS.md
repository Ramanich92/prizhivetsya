# Требования к региональным hero-картинкам

Файлы региональных hero-картинок теперь лежат в `assets/` и называются по slug страницы: `assets/<slug-страницы>-hero.jpg`. Суффиксы `draft` и версионные хвосты в названиях больше не используются.

## Размеры

| Тип страницы | Размер файла | Соотношение | Пример для Белгородской области |
|---|---:|---:|---|
| Обзор региона | 1440 × 810 px | 16:9 | `assets/belgorodskaya-oblast-hero.jpg` |
| Зона / подрегион | 1360 × 765 px | 16:9 | `assets/belgorodskaya-oblast-central-hero.jpg` |
| Зона / подрегион | 1360 × 765 px | 16:9 | `assets/belgorodskaya-oblast-north-forest-steppe-hero.jpg` |
| Зона / подрегион | 1360 × 765 px | 16:9 | `assets/belgorodskaya-oblast-south-steppe-hero.jpg` |

## Визуальные требования

- Формат: JPG, можно WebP только после одновременной правки ссылок в HTML.
- Вес после оптимизации: ориентир 120–220 КБ на файл.
- Без текста, логотипов, водяных знаков, рамок и искусственных подписей внутри картинки.
- Стиль: мягкая фотореалистичная сцена региона, спокойный свет, естественная глубина, без «плоской» декоративной иллюстрации.
- Композиция: важные детали держать ближе к центру; края могут обрезаться на мобильных экранах из-за `object-fit: cover`.
- Для обзора региона картинка должна быть узнаваемой для области в целом; для зоны — отражать характер зоны: лесостепь, степь, долина, побережье, горы и т. п.

## Белгородская область как эталон качества

Для Белгородской области ориентир такой: узнаваемый, спокойный, фотореалистичный пейзаж Черноземья/лесостепи с глубиной, натуральным светом и сельскохозяйственным настроением. Никакого текста в изображении. Финальные файлы для замены:

- `assets/belgorodskaya-oblast-hero.jpg` — обзор региона, 1440 × 810 px.
- `assets/belgorodskaya-oblast-central-hero.jpg` — центральная зона, 1360 × 765 px.
- `assets/belgorodskaya-oblast-north-forest-steppe-hero.jpg` — северная лесостепь, 1360 × 765 px.
- `assets/belgorodskaya-oblast-south-steppe-hero.jpg` — южная степная зона, 1360 × 765 px.

## Поисковые алиасы населённых пунктов

В версии v114 добавлен отдельный справочник населённых пунктов для поиска по регионам.

- Основной файл данных: `data/location-data.json`, поле `searchPlaces` у каждого субъекта.
- Дублирующий манифест для проверки и редактирования: `REGION_SEARCH_SETTLEMENTS.json`.
- Минимум на регион: 15 населённых пунктов.
- Эти названия используются только для поиска и ведут на обзорную страницу соответствующего региона, если населённый пункт не был отдельно привязан к природной зоне.
- Для малонаселённых субъектов в список включены города, ПГТ, посёлки и крупные/значимые сёла.
- Для Москвы и Санкт-Петербурга список построен по городам/поселениям/крупным районам внутри субъекта, поскольку это города федерального значения.
