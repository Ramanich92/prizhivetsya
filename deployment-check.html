Проверка заливки v50

Ожидаемая версия: v50. Эта сборка нужна, чтобы поймать ситуацию, когда одни файлы обновляются, а index.html остаётся старым.

Что должно открываться

  • index.html — на первом экране должен быть бейдж “v50”.
  • index-v50.html — проверочный дубль главной.
  • version.html — должен показывать v50.
  • data/site-version.json — JSON должен содержать "version": "v50".
  • BUILD_VERSION.txt и V50_DEPLOY_TEST.txt — маркеры корня.

Если снова смесь версий

  1. В репозитории открыть корневой index.html и проверить, есть ли там текст v50.
  2. Проверить, что архив не распакован как папка внутри репозитория.
  3. Удалить старые файлы публикации и залить содержимое архива в корень.
  4. Дождаться завершения GitHub Pages workflow/deploy.
  5. Открыть index.html?check=v50 и version.html?check=v50.
