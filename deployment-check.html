Проверка заливки v52

После публикации эти файлы должны открываться в актуальной версии. Если часть страниц показывает старые v48/v50, значит файлы не перезаписались или отдаются из кеша.

Контрольные адреса

  • index.html — публичная главная без технической кнопки проверки заливки.
  • version.html — заголовок v52.
  • data/site-version.json — JSON с "version": "v52".
  • V52_DEPLOY_TEST.txt — строка V52_DEPLOY_TEST_OK.
  • contacts.html — без старого футера “Сборка v48”.

Если снова смесь

  1. Удалить старые файлы публикации.
  2. Распаковать содержимое архива в корень, не папку архива внутрь корня.
  3. Проверить, что в корне лежат index.html, version.html, styles.css и папка data.
  4. Дождаться завершения GitHub Pages deploy.
