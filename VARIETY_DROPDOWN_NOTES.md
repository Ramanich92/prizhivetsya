# Раскрывающиеся сорта культур

Пилотная реализация добавлена на страницу `regions/yaroslavskaya-oblast-central-volga.html`.

## Структура данных

Внутри `data-culture-json` у культуры можно добавить поле:

```json
"varieties": [
  {
    "name": "Название сорта или гибрида",
    "type": "тип / срок созревания",
    "recommendation": "Надёжно | Рекомендовано | С укрытием / уходом | Рискованно",
    "place": "где лучше выращивать",
    "timing": "сроки посева, посадки или сбора",
    "note": "короткий практический комментарий"
  }
]
```

## Поведение интерфейса

- Нажатие на название культуры раскрывает сорта строками прямо под культурой, в тех же колонках таблицы.
- Поиск по таблице учитывает названия сортов, типы и комментарии.
- Если у других страниц нет поля `varieties`, таблицы продолжают работать как раньше.

## Масштабирование

Для полного сайта лучше вынести сортовые подборки в отдельный справочник по ключу `region + zone + culture`, чтобы не раздувать HTML страниц. Пилот оставлен внутри страницы, потому что это минимально инвазивный способ проверить формат.

## Формат v116

Карточный вид заменён на табличный: каждая строка сорта использует те же колонки, что и культура — название, тип, рекомендация, где лучше, сроки и комментарий. Это визуально ближе к основной таблице и удобнее для быстрого сравнения сортов.

## v117: расширение пилотной зоны Ярославской области

Пилотная страница `regions/yaroslavskaya-oblast-central-volga.html` расширена до полноценной тестовой структуры:

- 40 культур в таблице зоны.
- 200 сортов и гибридов всего.
- Минимум 5 сортов / гибридов у каждой культуры.
- Счётчики рекомендаций на странице обновлены:
  - Надёжно — 22
  - Рекомендовано — 12
  - С укрытием / уходом — 3

Важно: сортовые подборки предназначены как садово-огородные рекомендации для интерфейса. Перед массовым переносом такой логики на все регионы желательно дополнительно сверять сорта с актуальным Госсортреестром и региональными зонами допуска.
